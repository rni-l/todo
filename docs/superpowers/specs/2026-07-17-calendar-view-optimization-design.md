# 日历视图自适应与吸顶优化设计

日期：2026-07-17  
状态：已确认

## 目标

- 日历页面使用浏览器完整可用高度，不再受桌面 `920px` 或移动端 `620px` 的最大高度限制。
- 页面标题、操作区、统计区持续可见；月份标题和星期栏在月列表滚动时吸顶。
- 月视图默认只显示当前月份日期，相邻月份日期可通过持久化设置重新显示。
- 保留连续月份加载、日期选择、新建任务日期默认值和任务拖拽改期能力。

## 布局设计

日历路由为主内容区域增加专用布局状态。日历主区域使用 `100dvh` 高度和三行网格：页面标题、统计区、占满剩余空间的日历区。标题和统计区位于月列表滚动容器外，因此始终留在视图顶部；只有日历区内部滚动。

`.calendar-month-scroll` 移除现有 `max-height`，改为填满日历区的可用高度。`.calendar-shell` 和相关网格行设置 `min-height: 0`，确保内部滚动容器能够正确收缩。移动端继续通过现有主内容上下内边距避开固定顶栏和底部导航。

每个月份增加一个吸顶头部，包含月份标题和星期栏。吸顶头部限制在自己的月份区段内，下一个月份到达顶部后自然替换上一个月份。头部使用不透明主题背景、边框和明确的层级，避免任务内容从文字下方透出。

移动端继续保留每月网格的横向滚动。星期栏和日期网格使用两个同宽的横向滚动区域，并在同一个月份内双向同步 `scrollLeft`，保证星期与日期列始终对齐；不同月份之间互不影响。

## 设置与数据契约

在 `AppSettings` 中新增布尔字段：

```ts
calendarShowAdjacentDays: boolean
```

设置默认值为 `false`，并由服务端设置标准化逻辑统一补齐。已有数据文件缺少该字段时自动迁移为 `false`，不提升数据版本，也不要求手工迁移。

在 `设置 -> 外观` 中增加“显示相邻月份日期”开关，通过现有 `/api/settings` 更新并即时生效。开启后恢复当前相邻月份日期的完整显示和交互；关闭是新旧用户的默认行为。

## 月份内容行为

月份日期模型继续保留完整的 5 或 6 周、7 列网格，以维持正确的星期对齐和稳定的单元格尺寸。

当 `calendarShowAdjacentDays` 为 `false` 时：

- 非本月日期渲染为空白占位单元格，不显示日期数字、任务数量、单日任务或范围任务。
- 空白占位不带日期选择、任务新建或拖放处理，也不暴露为可操作日期。
- 跨月范围任务只绘制与当前月份相交的部分，并在月份边界保留延续标记。
- 范围任务行数和日期单元格顶部留白使用同一套裁剪结果，避免出现多余空白或任务条重叠。

当该设置为 `true` 时，沿用当前行为：相邻月份日期、任务、选择和拖放能力全部显示。

周视图不受此设置影响。

## 兼容性与失败处理

- 保留现有 `IntersectionObserver` 月份懒加载和向前加载后的滚动锚点恢复逻辑。
- 保留单日任务只更新截止日期、范围任务整体平移的拖拽契约。
- 浏览器不支持 `IntersectionObserver` 时维持当前降级行为，不新增错误提示。
- 设置保存失败继续使用现有 API 错误和 toast 处理，不引入仅前端生效的临时状态。
- 不改变后端 API 路由、请求结构或数据版本；设置响应仅增加一个字段。

## 验证方案

- 存储测试：新数据默认关闭相邻月份日期；旧数据自动补齐为 `false`；设置可更新为 `true` 并持久化。
- 日期与范围测试：月网格仍保持 35 或 42 个位置；隐藏模式正确裁剪月初、月末及跨月范围任务。
- 回归测试：日期选择、新建任务默认日期、单日拖拽、范围任务整体平移、月份前后懒加载均保持正常。
- 桌面浏览器：在常规和大屏视口验证日历填满剩余高度、无固定高度留白，标题与统计持续可见，月份和星期栏正确吸顶。
- 移动浏览器：验证固定顶栏/底栏不遮挡日历，星期与日期在横向滚动后仍对齐，页面不存在横向溢出。
- 分别验证设置关闭和开启两种状态，并运行完整测试与前端生产构建。
